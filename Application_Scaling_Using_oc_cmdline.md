# Scale Parksmap applications using the oc CLI tool #
# Typically, an OpenShift application is intended to run code at web scale.This means that an application may be processing hundreds, if not thousands, of requests a minute. In this scenario, it’s not practical to expect one pod to handle such a workload #
# The better solution is to add more pods that have identical logic to the cluster #
